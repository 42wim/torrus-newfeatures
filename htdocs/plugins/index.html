Torrus Plugins Documentation

Torrus main distribution is accomplished with a number of plugins. Usually a plugin is created when a feature is too specific for generic use, or too big, or is just developed by an independent contributor. Plugin names and documentation follow in alphabetical order.
  • cbqos: Cisco Class-based QoS discovery and collection
    The complete solution for automated CISCO-CLASS-BASED-QOS-MIB statistics.
  • dummy: Template for new plugins
    You can use it as a starting point for your new plugins.
  • httpcollector: HTTP data collector
    Author: Christian Schnidrig. This plugin collects data from HTTP GET queries.
  • overview: Torrus User interface enchancement
    Author: Christian Schnidrig. This plugin adds an alternative user interface to Torrus. The main feature is that it allows for creation of overview pages showing counters from different parts of a tree.
  • randomcollector: Random collector
    This plugin is designed for testing purposes only. It generates load on your Torrus system by creating data out of random number generator.
